
SQL - Язык структурированных запросов (SQL) — это язык программирования для хранения и обработки информации в базе данных.

DDL - Язык определения данных (DDL) (язык описания данных) — это семейство компьютерных языков, используемое в компьютерных программах для описания структуры баз данных.

DML - Data Manipulation Language (DML) (язык управления (манипулирования) данными) — это семейство  языков, пользователями баз данных для получения, вставки, удаления или изменения данных в базах данных. 

Database - организованный набор структурированной информации или данных, обычно хранящихся в электронном виде в компьютерной системе.

Database management system - программное обеспечение для эффективного хранения, управления, извлечения и манипулирования данными.

Related table - это табличная функция, которая возвращает таблицу значений

Relational database -  тип базы данных, в которой данные организованы в строки и столбцы, которые вместе образуют таблицу, в которой точки данных связаны друг с другом.

Relationship -  установленные ассоциации между двумя или более таблицами.

Record - Тип записи — это составной тип данных, состоящий из одного или нескольких идентификаторов и соответствующих им типов данных.

Primary Key - столбец в таблице реляционной базы данных, уникальный для каждой записи


Foreign Key - связать данные одной таблицы с данными другой таблицы.


One-to-many relationship - одна запись в таблице может быть связана с одной или несколькими записями в другой таблице.


One-to-one relationship - связь между информацией в двух таблицах, где каждая запись в каждой таблице появляется только один раз


Query - Запрос


Report - Отчеты анализируют данные и предназначены для расчета и представления данных в более формальной форме.


Form - Форма


Data type / varchar, char, int, decimal, binary, varbinary, text, blob, enum, set - Типы данных в sql


--------------------------
Constraints - Ограничения



NOT NULL - Ограничение NOT NULL используется для того, чтобы гарантировать, что данному столбцу таблицы никогда не будет присвоено нулевое значение.
UNIQUE - правило, согласно которому значения ключа действительны только в том случае, если они уникальны
PRIMARY KEY - столбец в таблице реляционной базы данных, уникальный для каждой записи
FOREIGN KEY - связать данные одной таблицы с данными другой таблицы.
CHECK - используется для обеспечения достоверности данных в базе данных и обеспечения целостности данных
DEFAULT - используется для установки значения по умолчанию для столбца
CREATE INDEX - INDEX используется для очень быстрого создания и извлечения данных из базы данных.
