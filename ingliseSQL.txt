
SQL - Structured query language-Язык структурированных запросов
DDL - Data Definition Language- язык описания данных
DML - Data Manipulation Language- язык управления (манипулирования) данными
Database - База данных
Database management system - Система управления базами данных
Related table - связанная таблица
Relational database - Реляционная база данных
Relationship - отношение, взаимоотношение; взаимосвязь, касательство, связь 
Record - записывать
Primary Key - Первичный ключ
Foreign Key - Внешний(Вторичный) ключ
One-to-many relationship - один-ко-многим
One-to-one relationship - один-к-одному
Query - Запрос
Report - представлять отчет {глаг.} отчитаться {возвр. глаг.}
Form - форма
Data type / varchar, char, int, decimal, binary, varbinary, text, blob, enum, set
VARCHAR: Это тип данных для хранения строк переменной длины.
CHAR: Это тип данных для хранения строк фиксированной длины.
INT: Это целочисленный тип данных для хранения целых чисел.
DECIMAL: Это тип данных, предназначенный для хранения чисел с плавающей запятой с фиксированной точностью.
BINARY: Это тип данных для хранения бинарных данных фиксированной длины.
VARBINARY: Это тип данных для хранения бинарных данных переменной длины.
TEXT: Это тип данных для хранения текстовых данных переменной длины.
BLOB: Это тип данных для хранения больших бинарных объектов (Binary Large Objects).
ENUM: Это тип данных, который ограничивает значение столбца на набор возможных значений, которые вы определяете при создании таблицы.
SET: Это тип данных, подобный ENUM, который также ограничивает значение столбца на набор возможных значений, но позволяет выбирать несколько значений из этого набора.
--------------------------
Constraints - можно назначить поле в качестве одного из следующих типов ограничений: Чтобы указать поле в качестве уникального ключа, можно использовать зарезервированное слово UNIQUE.
NOT NULL - позволяет определить, что значение в столбце не может быть NULL
UNIQUE - Возвращает уникальные строки в указанном диапазоне, убирая дубликаты.
PRIMARY KEY - позволяет однозначно идентифицировать каждую запись в ней.
FOREIGN KEY - помогает сохранять согласованность базы данных путем обеспечения так называемой «ссылочной целостности»
CHECK - позволяет задать некое логическое условие, которое должно быть истинным (TRUE) при вставке или обновлении данных в таблице.
DEFAULT - Для задания значения по умолчанию
CREATE INDEX - создаёт индексы по указанному столбцу(ам) заданного отношения, которым может быть таблица или материализованное представление.
